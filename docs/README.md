# Documentation

This folder contains supporting documentation that is useful for development history, review, and future refactors, but is not needed at the repository root.

## Contents

```text
docs/
  notes/            Historical implementation notes, hotfix notes, and design writeups
  patches/          Patch payloads and structured patch metadata
  refactor/         Refactor planning, validation artifacts, and baseline records
  visual_options/   Reference visualisation examples and montage images
```

The root `README.md` is the user-facing usage guide. Keep long-form implementation notes and historical analysis here so the repository root stays focused on installation, usage, and package metadata.
